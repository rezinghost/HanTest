Hanlab实验室主页管理指北

前言

实验室网页迁移到 CQB 网站统一管理后，使用的是一套叫博达的网站建设系统，强烈不建议浪费太多时间学他这一套（除非打算毕业去用这套系统的高校当网站运维…），首先不是什么广泛使用的系统，其次一些迁移前后的代码协调起来也很麻烦，正常情况下需要做的就更新实验室成员页面、添加发表文章以及上传相册就足够了。

如果需要改变页面结构或者什么地方出了问题，先学习 HTML，CSS，Javascript 的语法，博达系统相关的可以参考博达的教程网站，组件的语法初级、中级和高级。

登录后记得选择 hanlab ，另外两个定量生物学中心和定量生物学中心英文是韩老师在 CQB 网站内科研队伍页面的中文和英文介绍。上方导航栏选择的网站内容是编辑资料库的地方，而网站建设则既可以编辑资料库内容，也可以编辑模板和栏目设定。

总的来说网站由模板定义基本的页面结构和使用的网页组件，栏目定义特定网址访问的是那个模板生成的网页（如 cqb.pku.edu.cn/hanlab/Team.htm这个网址访问的是 Team 和 TeamMember 这两个模板生成的网页）以及每个网页组件使用什么资料库作为生成来源，资料库存放具体的内容。

修改了网站的模板和栏目之后需要点击左下角的发布网站，否则只能通过预览网站查看更改。而一般管理者需要做到的的网站更新，只需要在资料库中添加新的文章即可。

添加发表文章

添加实验室新发表的文章只需在 Publication 资料库中点击新增文章并在编辑完成点击上面的发布文章按钮。但是由于使用了之前版本网页中标出作者名和杂志名的 Javascript 脚本，因此添加内容时需注意符合特定格式。

  1. 标题无用处，但一般采用发表文章全名。
  2. 跳转链接勾选后填入在线预览网址，统一使用 doi 链接避免某些期刊网站改版导致的链接失效。
  3. 内容摘要取消勾选自动提取，填入特定格式的句子，需符合以下要求。
    1. 开头为作者名，以英文逗号加空格分隔’, ‘，最后两名作者之间可使用 ‘ and ‘ 分隔。作者名应为全名，以发表文章 pdf 内的格式为准。共同第一作者使用 ^ 标记，通讯作者使用 * 标记。列完全部作者名后使用英文句号作为结尾并空一格。
    2. 文章全名，使用英文句号作为结尾并空一格。文章名字中可以添加如斜体 <i></i> 和 上标 <sup></sup>等 html 标记，但是加粗标记以及加粗斜体联用标记将被删除。
    3. 发表杂志全名，使用英文逗号作为结尾并空一格，杂志全名中不得包含英文句号。
    4. 3 个字母月份加 4 位数字年份的 online 时间，以英文句号作为结尾。
  4. 附件为发表文章的 pdf 格式文件，其命名为第一作者、杂志全名、发表年份，以英文句号连接。
    1. 如有额外文件如勘误或补充文件，需注意附件的顺序，第一个附件将被认为是文章。
    2. 在勘误文件名称的年份后添加 Errata 后缀，参考格式为 XinyiWang.GenomeResearch.2017.Errata.pdf。更多勘误文件在 Errata 后额外添加数字，如 Errata2.pdf 等待。
    3. 如果有第一作者在一年内在同一杂志上发表多篇文章，可考虑在名字后加数字后缀。
  5. 右上角的发表时间，填写 online 时间，日期和时间出于惯例和强迫症选1 日 12 时。清空活动时间。

附使用到的正则表达式，方便出错时查询。

  1. 作者名查询和加粗
  2. 杂志名查询
  3. 加粗和加粗斜体联用标记的清除
  4. 年份判断

添加新实验室成员介绍

添加实验室成员介绍需要在 Team 下的各个资料库如 Staff 和 Post doctorates 等添加新页面。标题为成员名字，需要与发表文章条目中的名字相同，如果有多个不同格式名字，勾选副标题，将包括主名字在内的所有名字填入，否则无法自动加粗发表文章列表中的作者名。

由于使用了自动从 Publication 资料库中查找该成员发表的文章并插入的脚本，且需要保持不同页面之间样式的一致性，并不建议使用发表文章中的编辑功能，而是使用如下模板，填入成员相应内容后，在发表文章页面左上角点击切换成源代码，复制粘贴后使用。

各个区域内容的补充说明

复制模板到源码编辑模式后，再点击一下源码按钮切换回编辑模式，光标移到最下方后点上面的上传图片按钮，上传图片，这里图片会自动居中，需要切换回源码模式改一下。

改成

编辑完成后注意一下右上角的发表时间，Team 页面的左侧导航栏是按照发表时间从以前到现在这个顺序排的（和资料库中的显示顺序相反），未毕业的按进入实验室的时间填写，已经毕业的按照毕业的时间填写。

这里很多要求其实只是为了保持前后风格统一的强迫症，如果有个性化需求的其实也可以按照自己的喜好编写任意形式的内容，唯一需要注意的是自动插入实验室发表文献的脚本需要一个 <h2>Publication</h2> 的标题，和下面 <ol> 里的发表文献，如果没有会在 <h2>Personal Show</h2> 前创建一个。

实验室成员状态变更（离开实验室）

  1. 实验室内的定位需改成 Alumnus 或者 Alumna（也许删掉更好？）。
  2. 在经历的实验室中把 present 改成毕业的年月，如果是 Ph.D. student 改成 Ph.D.。
  3. 右上角的发表时间改成毕业时间。
  4. 从原本的资料库中剪切，粘贴移动到 Alumni 资料库中。
  5. Alumni 资料库选中粘贴过去的项目，点击发布（默认是不发布的）。
